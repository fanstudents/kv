CATEGORY資安與部署9 題
我們公司沒有工程師，可以導入嗎？

可以。部署由我們全程處理（線上或到府），日常使用只需要會用後台介面——像用 LINE 一樣簡單。教育訓練包含在部署費裡，教到會為止。

我該選「資料存我們這裡」還是「資料存你那裡」？

想最快上線、公司沒有 IT 部門管維運，選代管（Mode A）最省事；有 IT／資安部門、對資料主權要求高或需要通過內部稽核，選自建（Mode B）。兩種模式功能完全相同、定價相同，不確定的話我們會在諮詢時幫你判斷，之後也可以互相搬遷。

資料真的不會外流嗎？

看你選的模式。自建模式（Mode B）下，系統與資料庫都建在你指定的環境，對話與客戶資料完全不經過我們的伺服器存放；代管模式（Mode A）下，資料加密儲存於 TBR 受控環境，獨立於其他客戶，不會互相存取。兩種模式都搭配權限分級與完整稽核紀錄，每一筆存取都有跡可循。

系統部署在哪裡？地端主機或我們自己的雲端帳號都可以嗎？

可以，這就是「自建模式」——部署在公司內部的地端主機、NAS，或是你自己申請的雲端帳號（AWS、GCP、自架主機等）。我們負責安裝與串接，但主機的所有權與存取權限一直在你手上。如果你不想自己管主機，也可以選代管模式，由我們負責維運。

我們是不是跟其他客戶共用同一套系統？資料會不會被看到？

不會。無論選代管或自建，每一套 Agent Team（包含超級 Agent）都是獨立建置的一份環境與資料庫，不是多租戶共用的雲端服務。你的資料庫、對話紀錄與設定只屬於你，不會被其他客戶存取。

部署會不會動到我們現有系統，造成當機或資料遺失風險？

不會。Agent Team 以「旁掛」方式串接你現有的 LINE、官網、訂單或報表系統（多半透過官方 API 或 Webhook），不會修改或取代你原本的系統，也不需要停機部署。

我們自己也有 IT／資安部門，導入時需要他們配合什麼？

需要的通常只有：確認交付模式與部署環境（地端、你的雲端帳號，或我們代管）、開通必要的 API／Webhook 權限、確認防火牆規則。我們會提供部署架構文件，方便你的資安團隊事先審查。

有沒有存取紀錄或稽核機制，可以提供給我們做資安稽核？

有。後台的「任務與活動紀錄」會記錄每一次 Agent 執行的時間、內容摘要與結果；權限分級功能則記錄誰能看對話、誰能改設定。這些紀錄都可匯出供內部稽核使用，不因交付模式而不同。

如果之後想終止合作，資料會怎麼處理？

自建模式下，資料庫本來就在你的環境裡，終止合作時資料原地留在你手上，不需要「討回來」；代管模式下，我們會協助將資料完整匯出交還給你，並在確認交接後刪除代管環境中的副本。兩種模式都會協助移除相關串接設定。

CATEGORY場景與適用性6 題
我的產業比較小眾／傳統，Agent 適合我們嗎？

大部分適合。只要你的公司有「重複發生、規則清楚」的溝通或行政工作（回覆訊息、追訂單、發提醒、整理報表），就有 Agent 可以接手。產業小眾沒關係，我們可以用場景配對幫你判斷，或客製專屬 Agent。

可以只導入一部分功能，不要整套上線嗎？

可以，這也是多數客戶的做法。一位起聘，先解決最痛的那件事（通常是客服或訂單），看到成效後再逐步增聘其他部門的 Agent。

20 種 Agent 可以同時啟用嗎？會不會太複雜、沒人管得動？

可以同時啟用，而且這正是後台存在的原因——團隊總覽、任務紀錄、AI 成本儀表板都在同一個地方，管理 20 位 Agent 跟管理 3 位一樣簡單，不需要另外學新工具。

我們沒有 LINE 官方帳號或官網，也能導入嗎？

可以。多數 Agent 會建議搭配 LINE OA 或官網使用效果最好，但像對帳、報表、履歷篩選、翻譯這類 Agent 不依賴這些管道，也能獨立運作。部署前我們會先盤點你現有的系統再建議編制。

我該選「一般 Agent」還是「超級 Agent」？

如果你要解決的是明確的單一職務（例如客服回覆、訂單通知），一般 Agent（通用型／專業型）就足夠。如果你要的是「整個部門的營運策略＋執行」（例如電商全鏈路操盤、內容經營），超級 Agent 的真人經理人帶隊模式會更適合。不確定的話，場景配對會幫你判斷。

我們現有系統比較舊（甚至還在用 Excel、紙本），能接得上嗎？

能。多數情況下 Agent 是從「新的溝通管道」開始接手（例如 LINE 客服、Email 提醒），不強制要求你先做系統轉型。若確實需要資料整合，我們會在部署前評估並告知可行的串接方式。
